# 30-дневный план развития агентной системы генерации контента для 316.watch

## Неделя 1. Доработка MVP и стабилизация
**Цель:** сделать так, чтобы первый артикул обрабатывался быстро, точно и в нужном стиле.

- День 1–2: доработать промпты на основе первого теста (Tudor Monarch). Уточнить Tone of Voice, форматы полей, правила описания.
- День 3–4: протестировать 5–10 разных артикулов (разные бренды, новые и б/у, разные усложнения). Собрать обратную связь.
- День 5: внедрить чек-лист качества: какие поля проверять, какие факты перепроверять, типичные ошибки.
- День 6–7: зафиксировать финальную версию шаблона карточки, промптов и ToV. Сохранить как baseline.

## Неделя 2. Массовая обработка и автоматизация поиска
**Цель:** научиться обрабатывать пачки артикулов и сократить ручные шаги.

- День 8–9: доработать `run.py` — массовый запуск из CSV, создание единого Excel-файла на 10–200 позиций.
- День 10–11: улучшить поиск характеристик. Добавить WebFetch/WebSearch + базовый парсинг. Если сайты защищены — подготовить Playwright.
- День 12–13: внедрить fact-checker как обязательный шаг с возвратом на доработку.
- День 14: протестировать массовый импорт на 20–50 артикулах и проверить Excel на корректность.

## Неделя 3. Расширение контента и интеграция
**Цель:** добавить больше форматов контента и подготовить интеграцию с сайтом.

- День 15–16: добавить генерацию:
  - alt-текстов для изображений;
  - коротких анонсов для главной страницы;
  - писем/рассылок о новинках.
- День 17–18: изучить API/формат импорта Битрикс + Tilda. Подготовить маппинг полей Excel → CMS.
- День 19–20: автоматизировать подготовку папки с контентом для передачи в CMS (структурированный архив).
- День 21: тестовый импорт 10 карточек вручную через Excel, проверка отображения на сайте.

## Неделя 4. Публикация в соцсети, масштабирование и подготовка к ювелирным изделиям
**Цель:** сократить ручную работу при публикации, подготовить систему к регулярной нагрузке и заложить фундамент для работы с ювелирными изделиями.

- День 22–24: подготовить черновики постов к публикации:
  - шаблоны обложек/макетов для Дзен;
  - готовые тексты для копирования в ВК и Telegram.
- День 25–26: исследовать API ВКонтакте и Telegram (боты). Подготовить черновики для автопостинга при появлении доступов.
- День 27–28: создать базу знаний: FAQ по брендам, типичным ошибкам, списку проверенных источников.
- День 29–30: финальный обзор: метрики (время на артикул, точность, количество ручных правок), план доработок на следующий месяц, презентация результатов.

## Фаза 2. Расширение на ювелирные изделия
**Цель:** после отработки процесса для часов (новых и с пробегом) подготовить такой же конвейер для категории ювелирных изделий.

### Этап 1. Анализ и подготовка
- Собрать примеры карточек ювелирных изделий на сайте 316.watch.
- Определить структуру характеристик: вид изделия, металл, проба, вставки, вес, размер, сертификат, бренд, коллекция.
- Собрать реальные посты и статьи по ювелирной тематике (уже частично есть в `data/blog_articles.txt`).
- Создать отдельный шаблон карточки: `templates/jewelry_card_template.json`.

### Этап 2. Промпты и ToV
- Написать промпты для поиска характеристик ювелирных изделий.
- Определить источники: официальные сайты брендов (Damiani, Alrosa Diamonds, Поднебесные), каталоги, сертификаты.
- Дополнить `data/tov_social.md` разделом по ювелирной стилистике.

### Этап 3. Конвейер для ювелирки
- Адаптировать `run.py` для выбора категории: часы / часы с пробегом / ювелирные изделия.
- Настроить уточнения для ювелирки: вставки, размеры, сертификаты, комплектация, цена.
- Подготовить генерацию описаний, SEO-статей и постов в ювелирном стиле 316.watch.

### Этап 4. Тестирование
- Прогнать 3–5 реальных ювелирных артикулов.
- Сравнить результаты с существующими карточками на сайте.
- Доработать шаблон и промпты по обратной связи.

### Этап 5. Интеграция
- Добавить ювелирные карточки в единый Excel-экспорт.
- Подготовить отдельные посты для ювелирной линейки во всех соцсетях.

---

## Ключевые метрики на 30-й день
- Время подготовки пакета для 1 артикула: не более 10–15 минут ручной работы.
- Точность технических характеристик: > 95% без ручных правок.
- Покрытие: работает для 5–7 основных брендов магазина.
- Массовая обработка: готовность к импорту 100–200 позиций одним Excel-файлом.
- Tone of Voice: стабильное соответствие реальным постам 316.watch.
