# Маршрут проекта: агентная система генерации контента для 316.watch

## Для кого проект

**Заказчик:** контент-маркетолог интернет-магазина премиальных часов и ювелирных украшений 316.watch.

**Боль, которую решает проект:**
- Ручное создание карточки товара, SEO-описания, статьи блога и постов для соцсетей занимает много времени.
- При добавлении 10–20 новых артикулов в неделю и массовых импортах 100–200 позиций при запуске новых брендов объём ручной работы критично растёт.
- В премиум-сегменте нельзя допускать ошибки в технических характеристиках: калибр, механизм, материалы, водозащита.

**Что важно заказчику:**
- Точность и проверяемость фактов.
- Единый премиальный Tone of Voice магазина.
- Контроль перед публикацией: никаких автопостов и автовыгрузок на сайт без одобрения.
- Гибкость: система должна работать для новых часов, часов с пробегом и в перспективе для ювелирных изделий.

---

## Какой сценарий должен работать

### Вход
Пользователь передаёт агенту артикул часов, например `M2639W1A0U-0001`.

### Шаг 1. Обязательные уточнения
Перед началом работы агент спрашивает:
1. Часы новые или с пробегом?
2. Какую сортировку / форму карточки указать?
3. Гарантия (для новых — берётся с официального сайта, для б/у — уточняется у пользователя).
4. Для часов с пробегом:
   - состояние безеля;
   - комплектация;
   - состояние браслета/ремня;
   - состояние корпуса;
   - состояние стекла;
   - техническое состояние механизма;
   - цена.

### Шаг 2. Поиск характеристик
Агент ищет данные по приоритету:
1. Официальный сайт бренда — основной источник.
2. Авторизованные дилеры и профильные часовые магазины — для подтверждения.
3. Специализированные сайты, включая watchbase.com — для калибровых данных: количество камней, запас хода, частота, автоподзавод.

### Шаг 3. Формирование карточки
Агент заполняет поля строго по шаблону `templates/watch_card_template.json`:
- название: Бренд + Коллекция + Артикул;
- артикул — заглавными буквами;
- водозащита — только в метрах;
- количество камней — только число;
- запас хода: `65 часов`, `72 часа`;
- полуколебания: `28800 кол./час`;
- автоподзавод: `да двунаправленный`, `да однонаправленный`, `нет`;
- описание: 10–20 предложений, художественный но информативный тон, с SEO-ключами, без упоминания состояния и комплектации;
- SEO-заголовок, SEO-описание, ключевые слова.

### Шаг 4. Проверка фактов
- Перепроверка по второму независимому источнику.
- Флаги `verified` / `partial` / `manual_check_required`.
- Перегенерация при обнаружении ошибок.

### Шаг 5. Tone of Voice
- Проверка по файлу `data/tov_social.md`, созданному на основе реальных постов и блога 316.watch.
- Убираются пафос, перехваливание, лишние эмодзи, маркетинговый шум.
- Соответствие платформе: ВК — список характеристик и адрес бутика, Telegram — хэштеги и клуб привилегий, Дзен — длинное повествование, блог — экспертные блоки.

### Шаг 6. Генерация контента
Создаётся полный пакет в папке `output/YYYY-MM-DD/Brand-Articul/`:
- `characteristics.json` — карточка товара;
- `blog_article.md` — SEO-статья;
- `social_posts.md` — черновики для ВКонтакте, Telegram, Дзен;
- `card_import.xlsx` и `card_import.csv` — файлы для импорта в CMS.

## Как проверить результат

### Проверка технической точности
- Сверить калибр, механизм, материалы, диаметр, толщину, водозащиту с официальным сайтом бренда.
- Проверить форматы: артикул заглавными, водозащита в метрах, запас хода согласован.
- Убедиться, что не заполнены неточные или выдуманные поля.

### Проверка соответствия процессу
- В карточке есть все поля, запрошенные пользователем для новых или б/у часов.
- Гарантия и цена соответствуют тому, что указал пользователь.
- Описание не содержит состояния и комплектации, если это требование сохраняется.

### Проверка Tone of Voice
- Сравнить стиль описания, статьи и постов с реальными материалами 316.watch.
- Убедиться, что нет пафосных фраз, лишних эмодзи, выдуманных фактов и пересказа очевидного.

### Проверка импортных файлов
- Открыть `card_import.xlsx` / `card_import.csv` и убедиться, что поля корректно разнесены по колонкам.
- Проверить, что файл можно импортировать в Битрикс / Tilda без ручной правки.

## Как будем презентовать

### Формат
Краткая демонстрация на одном реальном артикуле с пошаговым разбором результата. Без лишних вступлений, по делу.

### Структура презентации
1. **Проблема:** сколько времени уходит на ручную подготовку карточки + SEO + посты.
2. **Решение:** агентная система из 6 субагентов + оркестратор.
3. **Демонстрация на артикуле Tudor Monarch M2639W1A0U-0001:**
   - `characteristics.json` — заполненные поля, источники, статус проверки;
   - `blog_article.md` — структура и стиль SEO-статьи;
   - `social_posts.md` — разница между ВК, Telegram и Дзен;
   - `card_import.xlsx` — готовность к импорту.
4. **Контроль качества:** как работают fact-checker и ToV-редактор.
5. **Масштабирование:** запуск пачками через CSV.
6. **Планы:** расширение на ювелирные изделия, автопостинг в соцсети при появлении API-доступов.

### Что показать как результат
- Скорость: полный пакет для одного артикула получен за один запрос.
- Точность: характеристики взяты с официального сайта, калибровые данные проверены.
- Стиль: описание, статья и посты соответствуют Tone of Voice 316.watch.
- Готовность к работе: Excel-файл для импорта и черновики для ручной публикации.
