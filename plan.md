# План: агентная система генерации контента по артикулу часов

## Цель
По одному артикулу (или пачкой) получать готовый пакет контента:
1. Заполненный файл с характеристиками (Excel, 100–200 строк массово; поштучно — markdown/JSON для проверки).
2. SEO-описание товара.
3. Статья для блога (обзор/история/коллекция).
4. Черновики постов для ВК, Telegram, Дзен.

---

## Архитектура: 6 субагентов + оркестратор

### Субагент 1: Поиск и извлечение характеристик
**Задача:** по артикулу и бренду найти официальный/авторизованный источник и вытащить проверенные характеристики.

**Источники:**
- Официальный сайт бренда — основной источник.
- Авторизованные дилеры и профильные часовые магазины — для общих характеристик.
- Специализированные часовые сайты, например https://watchbase.com/ — для калибровых данных: количество камней, запас хода, частота (полуколебания), тип автоподзавода.

**Инструменты:**
- WebFetch / WebSearch — поиск страницы товара.
- Browser-автоматизация (если понадобится) — Playwright через Python-скрипт, если сайт защищён от простого парсинга.

**Важно:**
- Если источник не найден или данные противоречат — не выдумывать, а отмечать «не подтверждено».
- Сохраняет источник каждой характеристики.

---

### Субагент 2: Генератор карточки товара
**Задача:** из сырых характеристик сделать структурированную карточку.

**Выход:**
- Поля: артикул, бренд, коллекция, название, калибр, механизм, материал корпуса, материал браслета/ремешка, размер, водозащита, функции, цена (если есть), SEO-заголовок, SEO-описание, мета-теги.
- Файл: Excel (.xlsx) для массового импорта или markdown для поштучной проверки.

**Инструменты:**
- Claude Code + запись в Excel/CSV через Python (pandas/openpyxl).
- Проверка на противоречия и пропуски.

---

### Субагент 3: Проверяющий фактов (Fact-Checker)
**Задача:** перепроверить все технические и брендовые факты перед генерацией контента.

**Проверяет:**
- Характеристики по второму независимому источнику.
- Соответствие калибра, механизма, материалов заявленным брендом.
- Отсутствие галлюцинаций и неточностей.
- Ценовые/маркетинговые утверждения.

**Выход:**
- Список проверенных фактов с источниками.
- Флаги «не подтверждено» / «требует ручной проверки».
- Замечания для перегенерации, если нашёл ошибки.

---

### Субагент 4: Tone of Voice редактор
**Задача:** привести все тексты к единому премиальному стилю магазина 316.watch.

**Контролирует:**
- Тон: респектабельный, спокойный, уверенный.
- Избегает пафоса, перехваливания, маркетингового шума.
- Корректность терминов: «механизм», «калибр», «корпус», «браслет», «ремешок», «водозащита».
- Единообразие названий брендов и коллекций.
- Соответствие аудитории премиум-магазина.

**Выход:**
- Отредактированные версии SEO-описания, статьи и постов.
- Комментарии по изменениям.

---

### Субагент 5: SEO-статья в блог
**Задача:** на основе проверенных характеристик и отредактированного ToV написать SEO-статью.

**Формат:**
- H1, структура с H2-H3, вводная, основная часть, вывод, ключевые слова.
- Тон: премиум, для аудитории интернет-магазина часов.
- Мета: title, description.

---

### Субагент 6: Посты в соцсети
**Задача:** подготовить черновики постов.

**Выход:**
- ВКонтакте — более развёрнутый пост с эмодзи.
- Telegram — короткий, акцент на факт/история.
- Дзен — статья-пост в более нейтральном информативном стиле.
- Хэштеги, призыв к действия, если уместно.

---

## Приём заявки на артикул: обязательные уточнения

Перед началом обработки агент должен задать пользователю:

1. **Состояние часов:** новые или с пробегом?
2. **Сортировка / форма карточки:** какую сортировку и форму отображения указать?
3. **Гарантия:**
   - Если новые — берётся с официального сайта бренда.
   - Если с пробегом — уточнить у пользователя. Если пользователь отвечает «2 года от нас» — записывать как `2 года от 316.watch`.
4. **Если часы с пробегом — уточнить дополнительно:**
   - безель
   - комплектация
   - состояние браслета/ремня
   - состояние корпуса
   - стекло
   - техническое состояние механизма
   - цена (обязательно уточнять у пользователя)

## Структура карточки товара и правила форматирования

### Состав названия товара
`Бренд` + `Коллекция` + `Артикул`
- Бренд: с заглавной буквы.
- Коллекция: с заглавной буквы.
- Артикул: полностью заглавными буквами.

### Поля карточки и форматы

| Поле | Формат | Пример | Источник |
|---|---|---|---|
| `articul` | Артикул заглавными буквами | `116610LN` | пользователь |
| `name` | Бренд + Коллекция + Артикул | `Rolex Submariner 116610LN` | генерируется |
| `brand` | С заглавной буквы | `Rolex` | официальный сайт / авторизованный дилер |
| `collection` | С заглавной буквы | `Submariner` | официальный сайт / авторизованный дилер |
| `condition` | `Новые` / `С пробегом` | `Новые` | пользователь |
| `mechanism` | `Механический` или `Кварцевый` | `Механический` | официальный сайт |
| `caliber` | Калибр | `3135` | официальный сайт |
| `jewels` | Только число, без слов | `31` | официальный сайт |
| `power_reserve` | Число + `час`/`часа`/`часов` согласовано | `72 часа` | официальный сайт |
| `frequency` | `число кол./час` | `28800 кол./час` | официальный сайт |
| `auto_winding` | `да двунаправленный` / `да однонаправленный` / `нет` | `да двунаправленный` | официальный сайт |
| `additional_functions` | Из фиксированного списка | `календарь`, `хронограф` и т.д. | официальный сайт |
| `case_material` | Например `Сталь`, `Сталь с PVD покрытием` | `Сталь` | официальный сайт |
| `bracelet_strap_material` | Например `Сталь`, `Каучук` | `Сталь` | официальный сайт |
| `glass` | Например `Сапфировое` | `Сапфировое` | официальный сайт |
| `dial_color` | Основной цвет, без составных | `Черный` | официальный сайт |
| `water_resistance` | Число + `м` (только метры; переводить из других единиц) | `300 м` | официальный сайт |
| `diameter` | Число + `мм` | `40 мм` | официальный сайт |
| `thickness` | Число + `мм`, как на официальном сайте (с запятой или без) | `14,1 мм`, `14 мм` | официальный сайт |
| `country` | С заглавной буквы | `Швейцария` | официальный сайт |
| `warranty` | `X лет` или `X лет от 316.watch` | `5 лет` / `2 года от 316.watch` | пользователь / официальный сайт |
| `price` | Уточнять у пользователя | — | пользователь |
| `description` | 10–20 предложений с SEO-ключами | — | генерируется |
| `bezel` | Только для часов с пробегом | — | пользователь |
| `accessories` | Комплектация, только для часов с пробегом | — | пользователь |
| `bracelet_strap_condition` | Состояние браслета/ремня, только для часов с пробегом | — | пользователь |
| `case_condition` | Состояние корпуса, только для часов с пробегом | — | пользователь |
| `glass_condition` | Состояние стекла, только для часов с пробегом | — | пользователь |
| `movement_condition` | Техническое состояние механизма, только для часов с пробегом | — | пользователь |
| `seo_title` | SEO-заголовок | — | генерируется |
| `seo_description` | SEO-описание | — | генерируется |
| `meta_keywords` | Ключевые слова | — | генерируется |
| `source_url` | URL источника | — | официальный сайт |
| `confidence_status` | `verified` / `partial` / `manual_check_required` | — | fact-checker |

### Список дополнительных функций (фиксированный)
- будильник
- второй часовой пояс
- гелиевый клапан
- дайверский безель
- индикатор приливов и отливов
- индикатор день/ночь
- календарь
- компас
- логарифмическая линейка
- лунный календарь
- мировое время
- тахиметр
- хронограф
- хронометр

### Правила
- Не выдумывать характеристики. Если данных нет — отмечать `manual_check_required`.
- Водозащита только в метрах. При необходимости переводить: `10 bar = 100 м`, `1 ATM ≈ 10 м`.
- Цвет циферблата — только базовые цвета (`Черный`, `Синий`, `Белый`, `Серебристый`, `Зеленый`, `Коричневый` и т.д.), без составных типа «серобуромалиновый».
- Количество камней — только число.
- Гарантия: если пользователь уточнил «от нас» — писать `от 316.watch`.
- Цена для часов с пробегом — всегда уточнять у пользователя.

### Оркестратор (главный агент)
**Задача:**
- Принимает артикул/список артикулов.
- Сначала задаёт обязательные уточнения (состояние, сортировка, форма, гарантия, цена, поля для б/у).
- Запускает субагентов последовательно: поиск → карточка → fact-check → ToV → статья и посты.
- Собирает результаты в единую папку.
- Показывает план перед стартом (согласно CLAUDE.md).
- Если fact-checker находит ошибки — возвращает на доработку.
- Финальный отчёт: что проверено, что требует ручной проверки.

# План: расширение поиска по ритейлерам и добавление поиска по изображению

## Контекст
Система уже умеет искать часы по артикулу+бренду через официальные сайты, авторизованных дилеров и русские магазины (alltime.ru, watcheson.ru, bestwatch.ru). Однако для некоторых брендов (Union Glashütte и других) данных в этих источниках не хватает. Также пользователь хочет уметь начинать поиск с фотографии часов.

## Цели
1. **Расширить список ритейлеров** — только те, где продаются оригинальные часы.
2. **Отфильтровать подделки/копии** на уровне источников.
3. **Добавить поиск по изображению** как альтернативный способ определить модель и получить характеристики.

## Подход

### 1. Расширение списка ритейлеров
Добавить домены в два уровня доверия:
- **Tier 1 — авторизованные/крупные ритейлеры:**
  - alltime.ru, watcheson.ru, bestwatch.ru, aswatch.ru (уже есть)
  - chrono24.com, watchfinder.co.uk, the1916company.com, cortinawatch.com, caratco.com, thewatchsource.co.uk, uhren2000.de, carollinum.cz, thewatchagency.com (уже частично)
  - Русские: shop-watches.ru, timeshop.ru, montre24.ru, watches-shop.ru, 316.watch (собственный), mercury.ru (официальный дистрибьютор Rolex/Chopard), alltime.ru, watcheson.ru, bestwatch.ru
- **Tier 2 — допустимые площадки с гарантией подлинности:**
  - ebay.com только карточки с пометкой Authenticity Guarantee
  - amazon.com (официальные магазины брендов)

**Черный список (точно подделки/реплики):** aliexpress.com, dhgate.com, wish.com, any replica keywords in domain/path ("replica", "copy", "aaa", "fake", "knockoff", "replica-watch", "копия", "реплика", "подделка").

### 2. Проверка оригинальности источника
- Вести явный whitelist/blacklist доменов.
- Приоритет отдавать Tier 1; Tier 2 и неизвестные источники помечать `confidence_status=manual_check_required`.
- Для неизвестных доменов — лёгкие эвристики:
  - домен в чёрном списке → пропускаем;
  - на странице есть слова "replica", "копия", "AAA", "подделка" → пропускаем;
  - цена в 10 раз ниже рынка → флаг подозрения (не блокируем, но помечаем).
- Финальный источник сохранять в `source_url` и `source_tier`.

### 3. Поиск по изображению
**Технология:** бесплатный Yandex Images reverse search (upload + по URL), без API-ключа. Fallback — Bing Visual Search, если пользователь предоставит `BING_SEARCH_API_KEY` в `.env`.

**Функционал:**
- Новый модуль `app/image_search.py`:
  - `yandex_reverse_search(image_path)` — загружает файл, возвращает список источников и текстовых сниппетов.
  - `yandex_reverse_search_by_url(image_url)` — то же самое для URL.
  - `extract_model_from_results(results)` — LLM/регулярки вытаскивает бренд+модель+артикул из заголовков/URL.
- Обновить `app/search.py`:
  - При неудаче текстового поиска можно подать найденную картинку с ритейлера в Yandex, чтобы найти официальную страницу.
- Обновить `app/app.py`:
  - Добавить загрузчик изображения. Если картинка загружена — сначала image search → определяем brand/articul → дальше обычный pipeline.
  - Показывать источники изображений и статус доверия.

### 4. Структура изменений
- `app/search.py`:
  - добавить `TRUSTED_DOMAINS`, `SUSPICIOUS_DOMAINS`, `FAKE_KEYWORDS`;
  - обновить `search_russian_retailers` и `_search_all` для фильтрации доменов;
  - добавить helper `_source_tier(url)`.
- `app/retailers.py` (новый):
  - каталог resolver-функций для новых магазинов;
  - whitelist/blacklist константы.
- `app/image_search.py` (новый):
  - Yandex reverse image search;
  - Bing Visual Search API wrapper (опционально, по ключу).
- `app/generator.py`:
  - принимать дополнительный параметр `image_path` и использовать image search результаты как дополнительный контекст;
  - добавить `source_tier` в карточку.
- `app/app.py`:
  - file_uploader для изображения;
  - отображение источника и tier.
- `requirements.txt`:
  - добавить `Pillow` (resize/validate images), `python-magic` (опционально MIME).

### 5. Тестирование
- Traser 109052 — должен по-прежнему находиться через alltime.ru.
- Luminox 3001 — должен находиться через watcheson.ru.
- Union Glashütte D007.444.16.033.00 — проверить, появится ли в новых ритейлерах; если нет — отмечаем manual_check.
- Cartier WSSA0037 — официальный/retailer tier.
- Загрузка фото часов (например, Traser) — image search должен определить модель.

### 6. Risks & mitigations
- Yandex image search может требовать CAPTCHA. Mitigation: realistic headers, retry, fallback на Bing API при наличии ключа, иначе manual_check.
- Новые ритейлеры могут менять структуру сайта. Mitigation: изолировать resolver-функции, логировать ошибки.
- Подделки всё ещё могут проскочить. Mitigation: строгий whitelist для источников, используемых в генерации контента; Tier 2 и unknown — только при явной пометке.

## Вопросы для уточнения

- Должен ли поиск по картинке быть основным способом ввода (загрузка фото → автоопределение) или fallback к текстовому поиску?

**Публикация:**
